# 2026-05-18 클라우드 배포 트러블슈팅 (8일차)

## 해결한 문제들

### 1. Vercel 새로고침 404
- **증상**: `/add` 등 React Router 경로에서 새로고침 시 404
- **원인**: Vercel이 SPA 라우팅을 모르고 정적 파일로 찾음
- **해결**: `vercel.json` 추가
  ```json
  { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
  ```

### 2. Gunicorn 미실행 (Application failed to respond)
- **증상**: Railway 서비스 green인데 "Application failed to respond"
- **원인**: Custom Start Command에서 `${PORT:-8000}` 미전개 → gunicorn 미시작
- **해결**: Start Command를 비워서 Dockerfile CMD로 fallback

### 3. CORS 차단
- **증상**: `No 'Access-Control-Allow-Origin' header is present`
- **원인 1**: `CORS_ALLOW_ALL_ORIGINS = DEBUG(=False)` → CORS_ALLOWED_ORIGINS 매칭 실패
- **해결**: `settings.py`에서 `CORS_ALLOW_ALL_ORIGINS = True`로 고정
  - 인증 없는 개인 앱이므로 보안 위험 없음
- **원인 2 (보조)**: `ALLOWED_HOSTS = *.railway.app` → Django 와일드카드 미지원
  - Django는 `*.` 미지원, `.railway.app`(점 시작)으로 변경

### 4. 포트 불일치 (최종 원인)
- **증상**: Gunicorn은 실행 중인데 여전히 응답 없음
- **원인**: Railway 재배포 시 PORT가 8000 → 8080으로 변경됐는데 도메인 설정은 8000 유지
- **확인**: Deploy Logs에서 `Listening at: http://0.0.0.0:8080` 확인
- **해결**: Railway → Settings → Networking 포트를 8080으로 변경

## 현재 상태

- Railway Django API: 정상 응답 (포트 8080)
- Vercel React: 정상 렌더링 + 새로고침 정상
- 와인 노트 CRUD: 정상 동작

## 현재 Start Command

비어있음 (Dockerfile CMD 사용):
```
sh -c "gunicorn --workers 2 --bind 0.0.0.0:${PORT:-8000} --access-logfile - --error-logfile - config.wsgi:application"
```

## 주의사항

- Railway 재배포 시 PORT가 변경될 수 있음
- PORT 변경 시 Settings → Networking 포트도 함께 업데이트 필요
- 장기적으로 `python manage.py migrate`를 Start Command에 포함시켜야 함
  (현재는 migrations이 이미 적용돼 있어서 문제없지만 스키마 변경 시 필요)
